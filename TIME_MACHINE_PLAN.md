# 計畫書:時光北科 — 重現過去一年的校園生活

## 一、核心概念

這是一個基於現有 2D RPG 校園地圖的擴充功能。玩家可以透過「時間軸控制器」設定一個過去的日期與時間流速，地圖上的 NPC、活動、人流密度會依照真實歷史資料動態重現——玩家「重新經歷」過去一年校園中曾經發生過的事件。

**比賽扣題**:
- 幸福科技: 校園生活、生活有感 (懷舊 / 情感連結)
- 中華電信: 資料分析、智慧生活

## 二、資料來源與蒐集方式

一次性爬取 (非即時)，目標來源:
- 學校官網活動公告 / 系所網站歷史公告
- 系學會、社團 Instagram / Facebook 公開貼文 (含日期、地點、活動描述)
- Dcard 北科大版熱門貼文 (篩選含時間地點資訊者)

每條資料整理為統一格式 (JSON):
```json
{
  "id": "evt_0001",
  "title": "資工系週年慶園遊會",
  "location": "活動中心",
  "location_id": "L023",
  "timestamp": "2025-04-12T11:00:00",
  "duration_minutes": 180,
  "popularity_score": 75,
  "source": "IG/資工系學會",
  "type": "活動"
}
```

`location_id` 需對應到現有地圖系統中已定義的座標節點。`popularity_score` 可由按讚/留言數簡單正規化得出 (0-100)。

爬蟲腳本一次性執行，輸出後**人工檢查清洗一輪**，確保地點對應正確、時間格式無誤，避免 demo 時出現資料錯誤。

## 三、系統架構與資料結構

### 1. 時間軸引擎 (核心)
- 玩家可設定「起始日期」與「時間流速」(例如 1 秒 = 1 分鐘 / 1 小時)
- 系統維護一個虛擬時鐘 `current_time`
- **資料結構: Min-Heap (優先佇列)**，以 `timestamp` 為 key，將所有事件存入 Heap
- 每個 tick，檢查 Heap 頂端事件是否 `timestamp <= current_time`，若是則 pop 並觸發該事件 (生成 NPC 公告/任務)
- 已觸發但尚在 `duration_minutes` 內的事件存入「進行中事件列表」，結束後從列表移除

### 2. 人流密度系統
- **資料結構: Segment Tree**，索引維度為「地點 × 時間區段 (以 30 分鐘為單位)」
- 預處理: 遍歷所有事件，將 `popularity_score` 依時間區段與地點累加進 Segment Tree
- 查詢: 虛擬時鐘變動時，對每個地圖節點查詢「當前時間區段的累積人流值」
- 依查詢結果決定該地點 NPC 數量 (例如: 0-20 → 1 個 NPC，20-50 → 3 個，50+ → 5 個並加上排隊 / 聚集動畫)

### 3. 地點映射
- **資料結構: Hash Map**，`location_id → 地圖座標 (x, y)`，沿用現有地圖系統已有的節點定義

## 四、玩法機制 (三項主要功能)

### 機制 1: NPC 公告
- 事件觸發時，該地點附近生成一個「公告 NPC」，玩家靠近按 E 互動，顯示該事件的標題、時間、來源摘要
- 視覺上以對話框呈現，語氣可用 AI (簡單 prompt 套版即可) 轉寫成 NPC 口吻，例如：「欸你聽說了嗎？今天活動中心有資工系的園遊會耶！」

### 機制 2: 限時任務
- 部分高 `popularity_score` 的事件額外生成任務：「請在 [事件開始時間] 前往 [地點]」
- 玩家若在虛擬時鐘到達該時間時人物位於該地點附近，任務判定完成，獲得獎勵 (經驗值 / 成就徽章)
- 若玩家錯過 (虛擬時鐘已超過 `timestamp + duration_minutes` 而未到達)，任務標記為「錯過」

### 機制 3: 人流重現 (地圖呼吸效果)
- 拖動時間軸時，地圖上各地點 NPC 數量即時依 Segment Tree 查詢結果增減，並有移動方向 (NPC 會「走向」高人流地點)
- 視覺上呈現「校園在這個時刻變得熱鬧 / 變得安靜」的動態感

### 機制 4 (收尾彩蛋): 錯過總結
- 玩家結束一段時間軸遊玩後 (例如快轉完一整天)，系統統計:
  - 完成任務數 / 總任務數
  - 「你錯過了這些活動：[列表]」
- 以較感性的文案呈現，呼應「幸福科技」的情感訴求——讓人意識到校園裡有多少自己錯過的精彩時刻

## 五、 UI / 技術整合

- 沿用現有 2D RPG 地圖介面，新增「時間軸控制條」(類似模擬市民的速度控制: 暫停 / 1x / 4x / 16x，搭配日期選擇器)
- 任務看板 (現有功能) 直接用於顯示機制 2 的限時任務列表
- NPC 公告以現有對話框系統呈現，內容為資料驅動而非預寫劇本

## 六、開發優先順序建議

1. **第一階段**: 資料爬取與清洗 (確保至少 30-50 筆高品質事件資料)
2. **第二階段**: Min-Heap 時間軸引擎 + NPC 公告機制 (機制 1) — 這是 MVP，單獨展示即可成立
3. **第三階段**: 限時任務 (機制 2) + Segment Tree 人流重現 (機制 3)
4. **第四階段 (若時間充裕)**: 錯過總結彩蛋 (機制 4) + 視覺細節打磨 (NPC 移動動畫、對話 AI 口吻生成)

## 七、Demo 腳本建議

1. 開場: 玩家設定時間軸回到「去年某個有大型活動的日期」，從早上開始播放
2. 快轉到活動開始前，觀察地圖上人流逐漸往活動中心聚集 (機制 3 視覺效果)
3. 玩家角色趕到現場，觸發 NPC 公告與限時任務完成 (機制 1+2)
4. 快轉到當天結束，顯示「錯過總結」畫面，作為情感收尾

---

這份計畫的核心優勢在於：**三個資料結構 (Min-Heap、Segment Tree、Hash Map) 都對應到明確且不可替代的功能**，資料蒐集為一次性、低風險，且整體敘事 (重現校園的一年) 具有情感共鳴與展示完整度。
