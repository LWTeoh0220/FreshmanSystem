# 📖 用戶故事 (User Story) & 劇本設定

## 1. 故事背景 (Background)
作為北科大僑生代表，我將在 7-8 月份陸續收到 115 學年來台留學的馬來西亞新生在 Line 群組上詢問各種行政、生活與課業問題。比起一遍又一遍重複同樣的叮嚀，我受此啟發，決定架設一個「大馬新生跨國生存與破關導航系統 (MY-NTUT Quest)」，透過經典 RPG 遊戲化 (Gamification) 的互動方式，向新生們提供全方位的資訊。

## 2. 核心價值與網頁優點 (Core Values)
1. **互動式知識獲取 (Interactive Learning)**：將繁雜的行政手續（簽證、門號、居留證等）轉化為「新手村主線任務」，讓新生在闖關中自然吸收資訊。
2. **極致沉浸的 UI/UX 體驗 (Immersive Experience)**：全面採用「經典 RPG 羊皮紙風格」，結合魔法陣、發光特效等視覺回饋，打破傳統行政網站的冰冷感，確保網頁的使用長久性與高黏著度。
3. **多樣化的遊戲體驗與激勵機制 (Gamified Incentives)**：
   - **動態探索**：3D 校園地圖遊玩（Three.js）、可互動的魔法卷軸與系統看板。
   - **成就系統與頭銜**：透過打怪升級（累積 EXP）、Minecraft 風格的成就進度樹，解鎖各類專屬頭銜（如：Lv.1 大馬萌新）。
   - **彩蛋與抽獎系統**：抽取北科大有趣的八卦、周邊美食指南、課程選擇攻略等次要生活資訊。

---

## 3. 目標用戶畫像 (User Personas)

### 🙍‍♂️ 萌新玩家：阿明 (大馬新生)
- **痛點**：第一次出國，對台灣的行政手續（居留證、開戶、選課）感到極度陌生與焦慮。看到校方滿滿是字的 PDF 文件就頭痛。
- **需求**：需要一個清晰、有明確先後順序的 Checklist，以及對校園空間的初步認知，最好能像玩遊戲一樣輕鬆。

### 🧙‍♂️ 老手玩家：學長姐 (協助者/僑生代表)
- **痛點**：每年都要回答一樣的問題，找不到一個快速且統一的資訊入口可以丟給學弟妹。
- **需求**：需要一個可以快速檢索特定問題（如：去哪裡辦電話卡）的工具，並能精準指路與分享。

## 4. 敏捷用戶故事 (Agile User Stories)

本專案將技術亮點（DAG、Min-Heap、Trie 等演算法）與玩家實際操作情境完美融合：

### Epic 1: 主線任務與破關導航 (Quest Tree - 基於 DAG 有向無環圖)
- **Story 1.1**：作為**大馬新生**，我希望能看到一個明確的「主線任務拓撲圖」（包含簽證、門號、ARC、銀行等），以便我了解每個行政手續的先後解鎖順序。
  - *驗收標準 (AC)*：任務間需有明確的鎖定/解鎖狀態；完成前置任務後，後續任務卡片會自動亮起並出現魔法流光特效。
- **Story 1.2**：作為**大馬新生**，如果不小心點錯任務狀態，我希望能有防呆的「撤銷機制」，以便我修正當前進度。
  - *驗收標準 (AC)*：系統需支援連鎖撤銷（基於 DFS 演算法），當取消父任務時，已解鎖的依賴子任務會自動恢復為鎖定狀態。

### Epic 2: 死線危機與戰情中心 (Emergency Dashboard - 基於 Min-Heap 最小堆積)
- **Story 2.1**：作為**大馬新生**，我希望系統能自動提醒我即將到期的任務（如：居留證辦理最後期限），以便我不會錯過重要死線而受罰。
  - *驗收標準 (AC)*：畫面右側（或世界樹看板）需有死線戰情室，利用 Min-Heap 演算法自動將最急迫的任務置頂，並呈現紅色警報。

### Epic 3: 快速傳送門引擎 (Fast Travel Search - 基於 Trie 字典樹 & Hash Map)
- **Story 3.1**：作為**學長姐或急需查資料的新生**，我希望能在搜尋框輸入關鍵字（支援簡繁體），就能立刻跳轉到對應步驟，以便省去一層層展開任務樹的時間。
  - *驗收標準 (AC)*：搜尋框需支援字首極速補完與同義詞映射（如輸入"簽證"自動匹配"ARC_Process"），提供極度順滑的檢索體驗。

### Epic 4: 3D 校園探索與虛擬空間 (3D Campus Navigator - 基於 Three.js)
- **Story 4.1**：作為**大馬新生**，我希望能在出發前先「逛」一下北科大校園，以便我對綜合大樓、餐廳等重要地標有初步的方位感知。
  - *驗收標準 (AC)*：提供 Low-poly 風格的 3D 校園地圖，支援沉浸式第一人稱與帶化身 (Avatar) 的第三人稱視角切換及平滑移動。

### Epic 5: 遊戲化成就與個人狀態 (Gamification & Progression)
- **Story 5.1**：作為**大馬新生**，我希望在完成特定行政手續後，能獲得成就徽章或經驗值升級，以便讓我有滿滿的破關成就感。
  - *驗收標準 (AC)*：頂部導覽列需顯示玩家等級（Lv）、經驗條（EXP）與金币。
  - *驗收標準 (AC)*：提供 Minecraft 風格的成就牆（Advancements Wall），達成里程碑時觸發特效回饋。
- **Story 5.2**：作為**大馬新生**，我希望能觸發一些互動彩蛋，以獲取北科八卦、美食等生活資訊。
  - *驗收標準 (AC)*：在特定任務節點或等級提升時，解鎖「遠古秘典」或「抽獎系統」，展示課外生活情報。

### Epic 6: 公會情報板與免登入連動 (Guild Intel Board)
- **Story 6.1**：作為**大馬新生**，我希望能在不用登入複雜學校系統的狀態下，就能收到針對我科系與宿舍的專屬情報，以便我不錯過重要公告。
  - *驗收標準 (AC)*：透過 LocalStorage 保存玩家的入籍設定（科系、宿舍），並在公會看板展示專屬的動態推播（模擬情報自動同步）。
