# PRD (Product Requirements Document)

## 🎯 產品目標 (Product Vision)
打造一個專為馬來西亞赴台新生設計的「跨國生存導航系統」。這不僅是一個靜態資訊網站，而是一個結合了「任務引導、快速搜尋、3D 空間感知與遊戲化成就」的全方位動態系統。目標是在期末專題中，以極高的完成度、技術深度與創意性脫穎而出。

## 👥 目標受眾 (Target Audience)
即將前往台灣的大學新生（特別是馬來西亞籍），對台灣的行政手續（簽證、門號、居留證、銀行開戶等）感到陌生與焦慮的使用者。

## ✨ 核心功能模組 (Core Modules)

### 1. 快速傳送門引擎 (Fast Travel Search Engine)
- **功能定位**：不再只是普通的搜尋框，而是專為老手設計的「捷徑」。
- **特色**：支援簡體與繁體中文（透過 `opencc-js` 轉換）。輸入關鍵字（如：户口、选课）可精準匹配並直接跳轉至對應的子流程。
- **UI/UX**：高質感的 iOS 模糊毛玻璃搜尋列。

### 2. 階層式任務拓撲圖 (Hierarchical Quest Tree)
- **功能定位**：新手專屬的主線引導。
- **巨觀視角 (Macro)**：顯示 5 大階段（簽證 -> 門號 -> ARC -> 銀行 -> 選課），並附帶「建議完成時間」。
- **微觀視角 (Micro / Drill-down)**：點擊巨觀階段可鑽取進入具體的子任務流程，任務間有嚴格的先後解鎖順序。
- **撤銷機制 (Undo)**：提供專屬撤銷按鈕，支援防呆的「連鎖骨牌撤銷」。

### 3. Minecraft 風格成就進度樹 (Advancements Wall)
- **功能定位**：視覺化的里程碑回饋。
- **特色**：採用類似 Minecraft 的格子 (Slots) 與科技樹設計。只有在完成「里程碑等級」的關鍵任務時才會點亮圖示。
- **互動**：Hover 時顯示遊戲風格的 Tooltip（名稱與條件），解鎖時有 Confetti 煙火特效。

### 4. 3D 校園室內外導航 (3D Campus Navigator)
- **功能定位**：建立對校園空間的初步感知。
- **特色**：提供低多邊形 (Low-poly) 的綜合大樓展示，允許自由移動探索（進入教室、餐廳）。
- **雙視角切換**：支援沉浸式第一人稱與帶有虛擬化身 (Avatar) 且平滑跟隨 (Lerping) 的第三人稱視角切換。

### 5. 免登入處室狀態綁定與自動化情報系統 (Guild Intel Board & Auto-Sync)
- **功能定位**：在不要求使用者登入學校系統的前提下，提供極具沉浸感的高密度個人化體驗。
- **技術架構 (規劃中)**：結合 GitHub Actions 每日定時爬蟲腳本，抓取學校各處室最新公告與行事曆，轉化為靜態 JSON。
- **極限模擬 (LocalStorage)**：透過「入籍儀式」讓新生登記科系與宿舍，利用瀏覽器緩存保存屬性，前端動態推播對應的專屬生存情報（如科系推薦課程、宿舍公告），達到真假難辨的整合效果。

## 🎨 UI/UX 設計規範 (Design Guidelines)
- **美學風格**：經典 RPG 沉浸式羊皮紙風格 (Classic RPG Parchment Style)。打破現代網頁的極簡冰冷感，全面引入遊戲化設計，包含復古邊框、羊皮紙底紋。
- **色彩計畫**：以復古牛皮紙色系 (#EEDC82、#D2B48C) 與深褐色文字 (#4A3B32) 為主色調，搭配魔法金 (進行中任務) 與復古朱紅 (通關戳章)。
- **遊戲化元件**：包含頂部玩家狀態列、打怪通關式的任務卡片、魔法流光進度線。詳見 `UI_UX.md`。
